# Security policy

## Supported versions

Security fixes are provided for the latest stable GitHub Release. Upgrade managed installations before
reporting a problem that is already fixed in a newer release.

## Report a vulnerability

Use GitHub's **Report a vulnerability** private reporting flow on the repository Security tab. Do not open
a public issue, discussion, or pull request with exploit details, credentials, private data, or an
unpatched proof of concept.

Include:

- the affected release or commit;
- the component and exact path;
- the impact and realistic exploit conditions;
- minimal reproduction steps; and
- any proposed mitigation.

You should receive an acknowledgement through GitHub. Disclosure timing will be coordinated in the private
advisory. Rotate or revoke any real credential that may have been exposed; never send it in the report.

## Scope

Reports about AI_ONBOARD's lifecycle manager, templates, validators, workflows, static site, and canonical
portable skills are in scope. Vulnerabilities in an upstream harness, service, or vendored dependency should
also be reported to that upstream maintainer; include the relevant upstream link when it affects this
package.
