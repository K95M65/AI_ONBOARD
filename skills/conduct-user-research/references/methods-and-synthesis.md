# Research methods and synthesis

Read the section that matches the research decision. Combine methods only when each answers a distinct
question.

## Select a method

| Need | Prefer | Watch for |
|---|---|---|
| Understand work, context, and mental models | Contextual inquiry or generative interview | Self-report replacing observation |
| Evaluate task completion and recovery | Moderated or unmoderated usability test | Tasks that reveal the intended control |
| Observe behavior over time | Diary study or longitudinal check-in | Participant fatigue and inconsistent entries |
| Measure a known pattern at scale | Survey with tested questions and sampling plan | Treating convenience samples as representative |
| Compare concepts before implementation | Concept evaluation | Visual polish overwhelming conceptual differences |
| Explain product telemetry | Targeted interview plus behavioral data | Assuming the event log explains intent |

Do not use a survey to discover an unknown problem space. Do not use preference voting to decide whether a
workflow is usable.

## Draft a session guide

Organize a guide around:

1. consent and expectations;
2. role, environment, and recent relevant experience;
3. a walkthrough of a real event or task;
4. focused probes about obstacles, information, handoffs, and consequences;
5. concept or prototype tasks, if applicable;
6. reflection and anything the study missed.

Ask “Tell me about the last time…” before “Would you use…”. Replace “Was that easy?” with “What did you
expect to happen?” and “What would you do next?”

## Sample deliberately

Recruit for behaviors and contexts tied to the question. Include important variation such as experience,
frequency, role, access needs, device, connectivity, risk, or workflow complexity. Set qualitative sample
size from expected diversity and saturation, not from a universal participant count.

## Synthesize

Use one or more of these structures:

- **Finding:** evidence pattern → context → consequence → confidence.
- **Journey:** trigger → steps → handoffs → obstacles → recovery → outcome.
- **Job:** situation → motivation → desired progress → constraints.
- **Usability issue:** task → observed breakdown → impact → frequency → likely cause.

Trace every conclusion back to evidence. Mark contradictions and possible moderator, sampling, novelty, or
prototype-fidelity effects.
