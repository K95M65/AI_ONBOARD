# Analytical frameworks

Frameworks are **lenses, not answers** — they organize evidence so implications surface. Pick the one that
fits the question; don't force all of them. The analysis leg of [`competitive-intel`](../competitive-intel/).

## Which framework for which question

See [`reference.md`](reference.md) for how to run each.

| Question | Framework |
|----------|-----------|
| Where do we stand? (quick) | **SWOT** |
| Is our advantage durable? | **VRIO** |
| How profitable is this industry, and why? | **Porter's Five Forces** |
| Where is value created / lost? | **Value chain** |
| Where should we invest across a portfolio? | **BCG growth-share** |
| What job do customers hire this for? | **Jobs-to-be-Done** |
| What macro forces are shifting? | **PESTEL** |

## Business-acumen habits

- **Follow the money.** Unit economics, margins, and *who captures the value* in the chain.
- **Second-order thinking.** "And then what?" — competitor response, substitution, customer adaptation.
- **Observation vs. inference.** State which is which, and attach a confidence level.
- **Quantify.** A ranged number ("15–20% share") beats an adjective ("significant").

## Output

**Insight → implication → recommended option(s)**, tied to the decision. A framework filled in for its own
sake is a worksheet, not analysis — the value is the "so what," not the completed grid.
