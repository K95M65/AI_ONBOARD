# Visual direction

Express the strategy through a coherent system rather than assembling fashionable fragments.

## Establish the concept

Choose a visual idea that reinforces the audience, outcome, and brand position. Describe it in one sentence,
then define:

- tone and emotional register;
- typography roles and hierarchy;
- color roles and contrast behavior;
- spacing rhythm and layout character;
- shape, border, depth, and surface language;
- image, illustration, icon, and data-visualization direction;
- motion purpose and reduced-motion fallback.

When an active design-picker workflow specifies an option count or sequence, follow it. Otherwise, when
collaboration is useful, present a small set of genuinely different directions with consequences, not minor
palette variants. Recommend one.

## Create the system

- Reuse the project's established design system when it exists.
- Define semantic tokens before styling individual sections.
- Limit type roles and use a deliberate scale.
- Reserve accent color for hierarchy and action instead of spreading it evenly.
- Establish container, grid, spacing, radius, border, shadow, focus, and motion tokens.
- Choose media that carries information or emotion the copy cannot provide alone.
- Design light/dark themes only when required; do not create a second theme as decoration.

## Avoid generic output

- Do not default to interchangeable gradients, glass cards, excessive pills, random blobs, or dashboard
  tiles without a strategic reason.
- Do not use every available visual effect.
- Do not sacrifice hierarchy, readability, or performance for novelty.
- Do not imitate a named living designer or copy a competitor's distinctive expression.

## Gate

Proceed when the direction can be explained from the strategy, the tokens can support all planned
components, and representative content remains legible and recognizable across narrow and wide layouts.
