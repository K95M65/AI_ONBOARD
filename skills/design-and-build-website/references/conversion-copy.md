# Conversion copy

Write for informed action, not pressure. Make the product easier to understand and trust.

## Build the message hierarchy

1. **Audience context:** Reflect the situation or problem in language the audience recognizes.
2. **Outcome:** State the specific improvement the offer enables.
3. **Mechanism:** Explain how it produces that outcome without hiding behind slogans.
4. **Difference:** Name a meaningful reason to choose it over the audience's current alternative.
5. **Proof:** Support important claims with real evidence.
6. **Objection handling:** Address cost, effort, risk, fit, trust, or switching concerns.
7. **Action:** Ask for the next appropriate commitment and explain what follows.

## Write section by section

- Give each section one message that advances the page's job.
- Lead headings with meaning; do not rely on the paragraph to rescue a vague headline.
- Prefer concrete nouns, active verbs, and observable outcomes.
- Translate features into audience consequences without overstating causality.
- Use specific calls to action such as "Book a 20-minute assessment" instead of context-free "Submit."
- Write labels, helper text, validation, confirmation, empty states, errors, and metadata as part of the
  experience, not as cleanup.
- Keep repeated CTA wording consistent unless the action or commitment changes.

## Protect credibility

- Never invent statistics, testimonials, customer logos, awards, security claims, guarantees, or scarcity.
- Mark unsupported proof as a content dependency.
- Preserve legal qualifiers when shortening regulated or contractual language.
- Distinguish aspiration, inference, and demonstrated result.
- Avoid manipulative countdowns, disguised advertisements, confirm-shaming, or inaccessible urgency.

## Refine

Read the page using only headlines, proof, and calls to action. Confirm that it tells one coherent story.
Then remove filler, duplicated promises, throat-clearing, and claims that could describe any competitor.
