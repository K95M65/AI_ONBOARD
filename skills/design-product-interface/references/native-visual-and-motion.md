# Native visual direction and motion

Create a recognizable product while preserving the platform signals users rely on.

## Establish native visual direction

- Start from the product role, use environment, information density, brand foundation, and platform
  conventions.
- Preserve system typography, symbols, control behavior, materials, selection, focus, and accessibility
  when customization would reduce recognition or integration.
- Express brand through content surfaces, hierarchy, illustration, data treatment, imagery, accent choices,
  and carefully selected custom components before restyling every system control.
- Align icons optically as well as geometrically. Match stroke, fill, corner, baseline, visual weight, and
  rendering size within an icon family.
- Test appearance changes, accent color, high contrast, transparency reduction, display scaling, dynamic
  type or text scaling, and grayscale where supported.

Avoid universal spacing, radius, shadow, or type rules detached from the target platform and task density.

## Use motion to explain change

Give each animation a job: preserve spatial continuity, connect cause and effect, reveal hierarchy, confirm
state change, or direct attention. Remove motion that merely delays frequent work.

- Match duration and distance to frequency: repeated commands should settle quickly; rare transitions may
  carry more explanation.
- Make long-running or decorative motion interruptible.
- Keep enter and exit behavior coherent without requiring symmetry.
- Animate properties the platform can render efficiently; profile before claiming a performance gain.
- Preserve state and focus when a transition is reduced or removed.
- Honor reduced-motion settings with a meaningful low-motion alternative, not simply a faster animation.

## Validate

Review real transitions at normal interaction speed and under rapid repeated input. Check cancellation,
interruption, window resizing, background/resume, input during animation, focus restoration, assistive
technology announcements, and reduced motion. Reject motion that obscures scope, delays correction, or
causes content to move without user control.
