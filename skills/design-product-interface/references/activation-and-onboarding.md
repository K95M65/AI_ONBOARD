# Activation and onboarding

Design first use around reaching meaningful value, not completing a tour.

## Define activation

- Name the first outcome that demonstrates the product's value to the target user.
- Distinguish account creation, setup completion, feature exposure, and genuine activation.
- Identify prerequisites, permissions, imported data, collaborators, or configuration required for value.
- Set an observable time-to-value measure and the point at which a user is considered stalled.

Do not select an activation event merely because it is easy to track. It should predict continued successful
use better than a page view or button click.

## Shape the first session

1. Remove setup that can be deferred, inferred, safely defaulted, or completed in context.
2. Let users perform a meaningful action with realistic examples or their own data.
3. Explain permissions at the moment their benefit is clear and preserve a useful denial path.
4. Reveal advanced concepts after the user has the context to understand them.
5. Preserve progress across interruption, authentication, permission prompts, and device changes.

Use a checklist only when the steps are genuinely independent and progress has lasting value. Use a guided
flow when order or validation matters. Use contextual education for capabilities best understood during
real work.

## Support different starting states

Account for invited users, administrators, returning users, migrated data, empty accounts, partial setup,
restricted roles, offline use, and users arriving through a deep link or shared object. Do not force a
single founder-style happy path on every role.

For stalled users, show what remains, why it matters, and a safe next action. Offer recovery from failed
imports, missing permissions, unavailable collaborators, invalid configuration, and abandoned setup.

## Validate

Observe whether representative users can:

- predict what the product will help them accomplish;
- reach first value without coaching;
- understand and recover from setup blockers;
- resume after interruption;
- discover the next meaningful workflow.

Pair activation rate and time-to-value with guardrails such as error, abandonment, support burden, unwanted
notifications, permission denial, and early retention. Avoid optimizing completion by hiding necessary
decisions or creating downstream failure.
